# System Runtime Layout

Этот каталог хранит runtime-состояние оркестрации задач.

## Структура

```text
system/
  state/
    global_state.json
    tasks/
      TASK_TEMPLATE.json
  logs/
  outputs/
```

## Назначение

- `state/` — каноническое состояние задач и общего execution-контура.
- `logs/` — журналы шагов оркестрации и проверок.
- `outputs/` — итоговые артефакты агентов, summaries, reports, deploy notes.

## Каноническая модель

- Если нужно понять, какие задачи активны, что у них за статус и кто владеет контекстом, смотреть в `system/state/`.
- Если нужно понять, что именно происходило по шагам, смотреть в `system/logs/`.
- Если нужно получить результат работы агента, смотреть в `system/outputs/`.
