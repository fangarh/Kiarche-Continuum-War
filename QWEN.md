# QWEN.md — Контекст проекта Kiarche Continuum War

## Обзор проекта

**Kiarche Continuum War** — научно-фантастическая 4X-стратегия с элементами RPG, разрабатываемая на Unity 6 для релиза в Early Access. Проект включает документационный веб-портал (`kcwweb/`), служащий информационным центром об игре.

### Сеттинг
Игра повествует о молодых расах, начинающих путешествовать через порталы и развиваться от железных мечей до огнестрела и заводов. Ключевые фракции включают Ки'Архе (распределённое поле сознания), Тал'Син (их последователи с механикой идентичности), а также различные молодые расы (Ре'Зиры, Ти'Оны, Ве'Ори, Родверы).

## Архитектура веб-приложения

### Data-Driven подход
Весь контент хранится в типизированных файлах данных под `src/data/`:

| Файл | Описание |
|------|----------|
| `navigation.ts` | Навигация по сайту |
| `home.ts` | Главная страница (hero, концепт-арты) |
| `overview.ts` | Описание игры, жанр, особенности |
| `lore.ts` | Лор: миры, история, фракции, персонажи |
| `mechanics.ts` | Игровые механики: core loop, бой, прогрессия |
| `architecture.ts` | Архитектура проекта для разработчиков |
| `conventions.ts` | Конвенции разработки |
| `documentation.ts` | Руководства, AI инструкции |
| `roadmap.ts` | Roadmap разработки (2026-2029) |
| `devlog.ts` | История разработки, решения |
| `team.ts` | Команда проекта |
| `index.ts` | Агрегатор всех данных |

Проект использует solution-style tsconfig:
- `tsconfig.json` — корневой файл с references
- `tsconfig.app.json` — приложение (строгий режим, `noUnusedLocals`, `noUnusedParameters`)
- `tsconfig.node.json` — Node-файлы (vite.config.ts и др.)

## Лор и механики игры

### Фракции

#### Играбельные
| Фракция | Описание |
|---------|----------|
| **Молодые расы** | Классическая 4X-кампания. Развитие через технологии и культуру. |
| **Тал'Син** | Уникальная механика идентичности. Синхронизация вместо прокачки. |

#### Неиграбельные (NPC)
| Фракция | Описание |
|---------|----------|
| **Синтекс** (Architects) | Создатели Portal Network. Скрытый конфликт с Ки'Архе. |
| **Ки'Архе** | Распределённое поле сознания. «Боги» за сценой. |
| **Этерны** (Древние) | Вознесённая раса. Нейтралитет/вражда с Ки'Архе. |
| **Кешари** (Асгарды) | Нейтралитет/союз с Ки'Архе. |
| **Сил'Ни** (Ноксы) | Пацифистская раса. Враги Ки'Архе. |
| **Вел'Кеты** (Гоа'улды) | Паразиты. Враги всех. |

#### Молодые расы (играбельные)
| Раса | Покровитель | Особенности |
|------|-------------|-------------|
| **Ре'Зиры** | Вел'Кеты | Нанитовый симбионт, орды, захват |
| **Ти'Оны** | Кешари | Телепатия, клонирование, тех. бонус |
| **Ве'Ори** | Этерны | Созерцание, наследие, ненасилие |
| **Родверы** | Независимые | Адаптивность, универсальность |

### Ключевые концепции

- **Ки'Архе** — распределённая когерентная структура, существующая как устойчивые узлы в многослойном пространстве
- **Тал'Син** — биологические последователи Ки'Архе с кремний-органической основой
- **Механика идентичности** — смерть допустима, если состояние зафиксировано и нет потери информации
- **Три состояния после смерти**: чистая интеграция, фрагментированная запись, эхо (аномалия)

## Важные заметки для работы с проектом

1. **Язык общения**: Всегда используй русский язык для ответов (см. `documentation/memory/user_language.md`).

2. **Data-driven архитектура (kcwweb)**: Никогда не хардкодь контент в компонентах. Все данные должны быть в `src/data/`.

3. **Стили (kcwweb)**: Каждая страница имеет свой CSS-файл. Избегай inline-стилей.

4. **Типизация (kcwweb)**: Все данные типизированы. Проверяй `src/types/` перед добавлением новых структур.

5. **Лор**: При работе с лором сверяйся с `documentation/memory/stargates_lore_discussion.md` — это единственный источник истины.

6. **Имена фракций** (переименовано для избежания авторских прав):
   | Оригинал | Новое имя |
   |----------|-----------|
   | Architects | Синтекс |
   | Goa'улды | Вел'Кеты |
   | Асгарды | Кешари |
   | Ноксы | Сил'Ни |
   | Древние | Этерны |

7. **Unity проект**: Использует Unity 6 (6000.4.0f1) с URP. Основные пакеты указаны в `Packages/manifest.json`.

8. **Структура кампании**: Две кампании на выбор — молодые расы (классическая 4X) и Тал'Син (уникальная механика идентичности).
