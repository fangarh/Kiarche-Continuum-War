# Улучшения системы поиска пути (Pathfinding Improvements)

**Дата:** 28 марта 2026 г.  
**Статус:** Готово к тестированию  
**Последние изменения:** 
- Исправлены ошибки компиляции (using, Raycast)
- Убрана зависимость от тега "Obstacle" (только компонент)
- Исправлено дергание/дрожание near destination:
  - `arrivalDistance` увеличен до 1.0
  - `stoppingDistance` увеличен до 0.5
  - Добавлен `minMoveDistance` (0.1) для предотвращения микро-движений
  - Добавлена отладка (логи, гизмо)
- Добавлена отладка в RTSInput (Debug.DrawLine, логи)

---

## Проблемы, которые были исправлены

### 1. Одиночные юниты застревали и не доходили до цели
**Причины:**
- Flow Field не учитывал препятствия динамически
- Отсутствие системы обнаружения застревания
- Примитивный raycast для обхода препятствий (только перпендикулярное направление)
- Нет физической коллизии с препятствиями

### 2. Группы юнитов проходили сквозь препятствия
**Причины:**
- Separation работал только между юнитами, но не с препятствиями
- Отсутствие CharacterController/Rigidbody для физической коллизии
- Formation игнорировала препятствия между юнитами и целью
- Нет multi-raycast для обнаружения препятствий

## Внесённые изменения

### 1. UnitPathfinder.cs — Улучшенное движение и избегание

#### Новые компоненты:
- **CharacterController** — добавлен для физической коллизии с препятствиями и другими юнитами
- **Система обнаружения застревания** — проверяет, двигается ли юнит, и пересчитывает путь при застревании
- **Multi-raycast obstacle avoidance** — веер из 5 лучей для обнаружения препятствий впереди

#### Улучшения:
- **CalculateObstacleAvoidance()** — новый метод для расчёта избегания препятствий через веер raycast
- **CharacterControllerMove()** — движение через CharacterController с обработкой столкновений
- **CheckIfStuck()** — проверка застревания каждые 0.5 сек с автоматическим пересчётом пути
- **RecalculatePath()** — небольшое смещение цели для обхода препятствия при застревании
- **Улучшенный CalculateSeparation()** — квадратичное затухание для более сильного отталкивания на близком расстоянии

#### Новые параметры (сериализуемые):
```csharp
[SerializeField] private float rotationSpeed = 10f;
[SerializeField] private float separationDistance = 1.5f;
[SerializeField] private float separationStrength = 3f;
[SerializeField] private float obstacleAvoidanceRadius = 1f;
[SerializeField] private float obstacleAvoidanceStrength = 5f;
[SerializeField] private int obstacleRayCount = 5;
[SerializeField] private float obstacleRayLength = 3f;
[SerializeField] private LayerMask obstacleLayerMask = -1;
```

### 2. FlowFieldManager.cs — Улучшенная генерация поля

#### Новые функции:
- **ScanObstacles()** — автоматическое сканирование препятствий на сцене через Physics.OverlapSphere
- **AddDynamicObstacle()** — добавление динамических препятствий (например, юнитов) в поле
- **FindNearestWalkableCell()** — поиск ближайшей проходимой ячейки, если цель в препятствии

#### Улучшения:
- **Автоматическое сканирование при старте** — все препятствия с тегом "Obstacle" или компонентом Obstacle автоматически регистрируются
- **Проверка достижимости цели** — если цель в препятствии, ищется ближайшая проходимая ячейка
- **Оптимизация хранения препятствий** — используется HashSet<Vector2Int> для быстрого доступа
- **Логирование** — добавлены Debug.Log для отладки генерации поля

#### Новые параметры:
```csharp
[SerializeField] private LayerMask obstacleLayerMask = -1;
[SerializeField] private bool autoScanObstacles = true;
[SerializeField] private float obstacleScanRadius = 50f;
```

---

### 3. Obstacle.cs — Динамическое обновление

#### Новые функции:
- **updateOnMove** — автоматическое обновление препятствия при перемещении
- **Update()** — проверка перемещения и вызов UpdateObstacle()

---

### 4. UnitController.cs — Улучшенная формация

#### Улучшения:
- **Случайное смещение** — небольшое случайное смещение (-0.2..0.2) для предотвращения наложения юнитов
- **Спиральная формация** — для групп >6 юнитов используется несколько кругов (спираль)
- **Оптимизированный радиус** — радиус формации уменьшен для маленьких групп (коэффициент 0.8)

## Настройка в Unity Editor

### 1. CharacterController для юнитов
CharacterController добавляется автоматически при старте через UnitPathfinder.Awake().

**Параметры по умолчанию:**
- Radius: 0.3
- Height: 1.0
- Center: (0, 0.5, 0)

### 2. Слой препятствий (Obstacle Layer)
1. Создайте слой "Obstacle" (Layer 8 или другой свободный)
2. Назначьте этот слой на все препятствия
3. В UnitPathfinder настройте `Obstacle Layer Mask` на этот слой
4. В FlowFieldManager настройте `Obstacle Layer Mask` на этот слой

### 3. Тег препятствий
Альтернативно используйте тег "Obstacle" для всех препятствий.

### 4. Компонент Obstacle
Добавьте компонент `Obstacle` на все статические препятствия:
- **Obstacle Radius:** радиус препятствия
- **Register On Start:** автоматически регистрировать при старте
- **Update On Move:** обновлять при перемещении (для динамических препятствий)

## Тестирование

### Сценарий 1: Одиночный юнит и препятствие
1. Поместите юнита и препятствие между ним и целевой точкой
2. Отдайте приказ о перемещении
3. **Ожидаемо:** Юнит должен обойти препятствие и достичь цели

### Сценарий 2: Группа юнитов и узкий проход
1. Выделите группу из 5-10 юнитов
2. Отдайте приказ о перемещении через узкий проход в препятствии
3. **Ожидаемо:** Юниты должны пройти через проход по очереди, не застревая

### Сценарий 3: Группа юнитов и препятствие
1. Выделите группу юнитов
2. Отдайте приказ о перемещении к точке за препятствием
3. **Ожидаемо:** Юниты должны обойти препятствие, сохраняя формацию

### Сценарий 4: Динамическое препятствие
1. Поместите движущееся препятствие на пути юнита
2. **Ожидаемо:** Юнит должен обнаружить и обойти движущееся препятствие

### Сценарий 5: Застревание
1. Создайте ситуацию, где юнит может застрять (например, между двумя препятствиями)
2. **Ожидаемо:** После 3 проверок застревания (1.5 сек) юнит должен пересчитать путь с новым смещением цели

### Сценарий 6: Большая группа (10+ юнитов)
1. Выделите большую группу юнитов
2. Отдайте приказ о перемещении
3. **Ожидаемо:** Юниты должны распределиться в спиральную формацию и не накладываться друг на друга

## Отладка

### Визуализация Flow Field
В FlowFieldManager включите `Draw Debug Gizmos` для визуализации:
- **Зелёные/жёлтые/красные точки** — стоимость ячейки (зелёный = близко к цели, красный = далеко)
- **Белые линии** — направления потока
- **Красные точки** — препятствия

### Визуализация Obstacle Avoidance
В UnitPathfinder raycast лучи отрисовываются красным цветом при обнаружении препятствия.

### Визуализация Separation
В UnitPathfinder включите отладку для отображения радиуса separation (жёлтая сфера).

### Логи
- FlowFieldManager выводит логи о генерации поля и найденных препятствиях
- Проверьте консоль на предупреждения о недостижимых целях

## Известные ограничения

1. **Динамические препятствия:** Flow Field не обновляется автоматически при перемещении препятствий. Требуется вызов `FlowFieldManager.ScanObstacles()` или ручное обновление через `SetObstacle()`.

2. **Производительность:** Multi-raycast (5 лучей на юнит) может быть затратным для больших армий (50+ юнитов). Рассмотрите возможность уменьшения `obstacleRayCount` или увеличения интервала проверки.

3. **CharacterController:** Может вызывать проблемы на неровном terrain. Убедитесь, что terrain имеет правильный слой и высоту.

4. **Застревание в углах:** Система пересчёта пути помогает, но не гарантирует 100% успех в сложных лабиринтах.
